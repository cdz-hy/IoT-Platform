## 三、数据库初始化步骤

### 3.1 确保 MySQL 服务已启动

打开"服务"（Win+R 输入 `services.msc`），找到 MySQL 服务并确保其状态为"正在运行"。

### 3.2 配置数据库连接

打开 `Java-backend/src/main/resources/application.yml`，修改数据库连接信息：

```yaml
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/iot_platform?useSSL=false&serverTimezone=Asia/Shanghai&characterEncoding=UTF-8&allowPublicKeyRetrieval=true&createDatabaseIfNotExist=true
    driver-class-name: com.mysql.cj.jdbc.Driver
    username: root
    password: 你的MySQL密码    # 修改此处
```

> **说明：** URL 中的 `createDatabaseIfNotExist=true` 会自动创建 `iot_platform` 数据库，无需手动建库。

### 3.3 自动初始化

项目配置了 `spring.sql.init`，后端启动时会自动执行：
- `schema.sql` — 创建15张数据表（如已存在则跳过）
- `data.sql` — 插入初始数据（使用 `INSERT IGNORE` 避免重复）

首次启动即可自动完成数据库初始化，无需手动执行 SQL。

### 3.4 手动初始化（可选）

如需手动初始化，可使用 MySQL 命令行或 Navicat：

```sql
-- 执行建表脚本
SOURCE E:/高质量编程/IoT Platform/Java-backend/src/main/resources/schema.sql;

-- 执行初始数据脚本
SOURCE E:/高质量编程/IoT Platform/Java-backend/src/main/resources/data.sql;
```

## 五、MQTT Broker 说明

### 5.1 内嵌 Broker

本项目使用 **Moquette 0.17** 作为内嵌 MQTT Broker，随后端 Spring Boot 一起启动，**无需额外安装**。

| 配置项 | 值 | 说明 |
|--------|-----|------|
| TCP 端口 | `1883` | MQTT 标准端口 |
| WebSocket 端口 | `8083` | 浏览器端 MQTT 连接 |
| WebSocket 路径 | `/mqtt` | 完整地址: `ws://localhost:8083/mqtt` |

### 5.2 MQTT 认证方式

设备连接时需填写：

| 字段 | 值 |
|------|-----|
| Client ID | 设备ID（如 `DEV_xxxxxxxx`） |
| Username | 设备ID（与 Client ID 相同） |
| Password | 设备密钥（注册设备时自动生成的 `secret`） |

认证由 `MqttBrokerConfig` 中的 `IAuthenticator` 实现，验证设备ID和密钥是否匹配数据库记录。

### 5.3 MQTT 主题规范

| 主题 | 方向 | 说明 |
|------|------|------|
| `iot/{productId}/{deviceId}/telemetry` | 设备→平台 | 上报遥测数据 |
| `iot/{productId}/{deviceId}/event` | 设备→平台 | 上报事件 |
| `iot/{productId}/{deviceId}/command/{commandId}` | 平台→设备 | 下发命令 |
| `iot/{productId}/{deviceId}/command/response` | 设备→平台 | 命令响应 |
| `iot/{productId}/{deviceId}/shadow/delta` | 平台→设备 | 影子差异下发 |

## 七、设备模拟器使用说明

### 7.1 使用平台内置模拟器（推荐）

平台左侧菜单"设备模拟器"提供基于 Web 的 MQTT 模拟器：

1. 选择已注册的设备
2. 点击"连接"
3. 点击"随机生成"生成符合物模型的数据
4. 点击"上报数据"
5. 可接收平台下发的命令并自动响应

### 7.2 使用 MQTT.fx 模拟器

#### 7.2.1 连接配置

打开 MQTT.fx，点击齿轮图标新建连接：

| 配置项 | 值 |
|--------|-----|
| Broker Address | `localhost` |
| Broker Port | `1883` |
| Client ID | 设备ID（如 `DEV_xxxxxxxx`） |
| User Name | 设备ID（同 Client ID） |
| Password | 设备密钥（注册时生成的 `secret`） |

点击 Connect，右上角指示灯变绿表示连接成功。

#### 7.2.2 订阅主题

在 Subscribe 选项卡中订阅：

```
iot/产品ID/设备ID/command/#
iot/产品ID/设备ID/shadow/delta
```

#### 7.2.3 上报遥测数据

Publish 选项卡：

| 字段 | 值 |
|------|-----|
| Topic | `iot/产品ID/设备ID/telemetry` |
| Payload | `{"temperature":28.5,"humidity":65}` |

#### 7.2.4 上报事件

| 字段 | 值 |
|------|-----|
| Topic | `iot/产品ID/设备ID/event` |
| Payload | `{"eventIdentifier":"highTemperature","eventType":"warn","temperature":42.5}` |

#### 7.2.5 命令响应

收到命令后，发布响应：

| 字段 | 值 |
|------|-----|
| Topic | `iot/产品ID/设备ID/command/response` |
| Payload | `{"commandId":"CMD_xxxxxxxx","status":"success","result":"执行成功"}` |

> **注意：** `commandId` 必须与收到的命令中的 `commandId` 一致。

## 九、常见问题

### Q1: 后端启动报错 "Unknown database 'iot_platform'"

**原因：** MySQL 中没有 `iot_platform` 数据库。

**解决：** 确认 `application.yml` 中 URL 包含 `createDatabaseIfNotExist=true`，或手动创建数据库：
```sql
CREATE DATABASE iot_platform DEFAULT CHARACTER SET utf8mb4;
```

### Q2: MQTT.fx 连接报错 "Bad user name or password"

**原因：** 认证信息不正确。

**解决：**
1. 确认 Client ID = Username = 设备ID（如 `DEV_xxxxxxxx`）
2. 确认 Password 是设备的 `secret`，不是管理员密码
3. 确认设备已通过平台注册

### Q3: 前端页面白屏或报错

**原因：** 后端未启动或 API 请求失败。

**解决：**
1. 确认后端已启动（http://localhost:8080/api/swagger-ui/index.html 可访问）
2. 确认前端 `vite.config.js` 中代理配置正确

### Q4: 遥测数据上报后看不到数据

**原因：** 属性标识符与物模型不匹配。

**解决：** 确认上报 JSON 的 key 与产品物模型中定义的 `identifier` 一致，如 `temperature`、`humidity`。

### Q5: 命令下发后一直显示"执行中"

**原因：** 设备未响应命令。

**解决：**
1. 确认设备在线
2. 确认设备已订阅命令主题 `iot/产品ID/设备ID/command/#`
3. 确认设备已发送响应到 `iot/产品ID/设备ID/command/response`
4. 超过5分钟未响应会自动标记为"超时"
