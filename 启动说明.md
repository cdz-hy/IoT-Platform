# IoT Platform 启动说明

## 环境要求

- JDK 1.8 或更高版本
- Maven 3.6+ (用于构建后端)
- Node.js 16+ (用于前端开发)
- MySQL 5.7+ (可选，默认使用H2内存数据库)

## 方式一：IDE中启动（推荐）

### 1. 后端启动（IntelliJ IDEA / Eclipse）

1. 打开IDE，导入 `Java-backend` 项目
2. 等待Maven依赖下载完成
3. 运行主类 `IotPlatformApplication.java`
4. 看到以下日志表示启动成功：
   ```
   IoT Platform 启动成功！
   API文档: http://localhost:8080/api/swagger-ui/
   MQTT Broker: tcp://localhost:1883
   ```

### 2. 前端启动

```bash
cd Vue-frontend
npm install    # 首次运行需要安装依赖
npm run dev
```

访问 http://localhost:5173

## 方式二：命令行启动

### 1. 配置Maven环境变量

将Maven的bin目录添加到系统PATH：
```
C:\Program Files\Apache\maven\bin
```

### 2. 构建并运行后端

```bash
cd Java-backend
mvn clean package -DskipTests
java -jar target/iot-platform-1.0.0.jar
```

## 使用MySQL数据库

### 1. 创建数据库

```sql
CREATE DATABASE iot_platform DEFAULT CHARACTER SET utf8mb4;
```

### 2. 修改配置

编辑 `application.yml`，注释掉H2配置，启用MySQL配置：

```yaml
spring:
  # MySQL配置
  datasource:
    url: jdbc:mysql://localhost:3306/iot_platform?useSSL=false&serverTimezone=Asia/Shanghai&characterEncoding=utf8mb4
    driver-class-name: com.mysql.cj.jdbc.Driver
    username: root
    password: 12344321
  jpa:
    database-platform: org.hibernate.dialect.MySQL8Dialect
    hibernate:
      ddl-auto: update
```

### 3. 执行初始化脚本（可选）

```bash
mysql -u root -p12344321 iot_platform < Java-backend/src/main/resources/schema.sql
mysql -u root -p12344321 iot_platform < Java-backend/src/main/resources/data.sql
```

## 常见问题

### 1. 端口被占用

```bash
# 查找占用端口的进程
netstat -ano | findstr :8080
# 结束进程
taskkill /PID <进程ID> /F
```

### 2. Maven依赖下载慢

在 `settings.xml` 中配置阿里云镜像：
```xml
<mirrors>
  <mirror>
    <id>aliyun</id>
    <mirrorOf>central</mirrorOf>
    <url>https://maven.aliyun.com/repository/public</url>
  </mirror>
</mirrors>
```

### 3. 前端启动报错

```bash
# 清除缓存重新安装
rm -rf node_modules package-lock.json
npm install
```
